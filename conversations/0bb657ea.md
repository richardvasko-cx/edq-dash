**User:** What's hops

**Gemini:** "Hops" refers to the number of individual mail servers (or "nodes") that an email passes through as it travels from the sender's infrastructure to the recipient's mailbox provider.

Each time a message is received by a server and then forwarded to the next, it counts as one "hop." You can see these recorded in the message headers as `Received` lines.

*   **Why it matters:** While 6 hops is well within the normal range for modern email delivery, an excessively high number of hops (typically 15+) can be a signal of misconfigured routing or mail forwarding loops. This can cause unnecessary latency and, in some cases, trigger spam filters that view high hop counts as an attempt to obfuscate the email's true origin.
*   **In this context:** Your 6-hop count is healthy and indicates a standard, direct path. It is not contributing to any deliverability issues you may be seeing.

**Next Step:** If you suspect routing issues despite the normal hop count, use the **MX Tool** in this dashboard to verify that the MX records for your sending domain are correctly configured and pointing to the expected mail exchangers.
