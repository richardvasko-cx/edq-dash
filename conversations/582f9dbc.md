# Data dictionary

> Each app simulation in Braze Pilot is instrumented to collect a variety of events and attributes based on user activity in the app. 

## The approach to data

The app logs custom attributes and events typical of the industry represented by the fictional brand. You can use these attributes to power demos for a variety of common use cases. 
Generally, all events and attributes are prefixed with a short code that corresponds to the app simulation responsible for the data. For example:

- All data logged by the Steppington app simulation are prefixed with `st_`
- All data logged by the PantsLabyrinth app simulation are prefixed with `pl_`
- All data logged by the MovieCanon app simulation are prefixed with `mc_`

**Gemini:** ### Summary
The Data Dictionary outlines the custom events and attributes logged by Braze Pilot app simulations, such as MovieCanon, Steppington, and PantsLabyrinth. This structured data allows teams to simulate realistic, industry-specific user behaviors to power targeted demo campaigns and testing scenarios.

### Key Best Practices
* **Apply Simulation Prefixes:** Always prepend custom data with the correct app-specific code (e.g., `mc_` for MovieCanon, `st_` for Steppington, or `pl_` for PantsLabyrinth) to ensure proper routing.
* **Leverage Nested Properties:** Utilize event-specific properties (such as `title` for the `mc_watched_movie` event) to build highly personalized message templates and dynamic segments.

### Common Pitfalls
* **Omitted or Mismatched Prefixes:** Omitting the required short-code prefix will prevent the simulation from correctly logging and mapping user actions to the target demo brand.
